# 3장. 좌표에 의미를 입히다 — 온톨로지의 등장

## 챕터 정보
- **위치**: 본문 3장 (에세이의 가장 핵심)
- **분량**: 약 700자
- **역할**: 에세이의 지적 정점. "왜 온톨로지인가"의 답을 명료하게 전달

## 핵심 메시지
> "드론들이 같은 좌표가 아니라 같은 세계관을 공유할 때, 비로소 군집은 '우리'가 된다."

이 한 문장이 에세이 전체의 thesis. 다른 모든 챕터가 이 문장을 향해 모이고 이 문장에서 출발해 4장·에필로그로 흩어진다.

## 작성 구조 제안

**1단락 — 온톨로지란 무엇인가 (약 150자)**
- 어원: 그리스어 ontos(존재) + logos(학문) — 원래는 철학 용어
- 컴퓨터 과학에서의 온톨로지: **"한 도메인의 개념과 관계를 형식화한 공유 사전"**
- 쉽게 말하면 "기계들이 같은 단어를 같은 뜻으로 쓰게 만드는 규칙집"
- 의료(SNOMED CT), 금융(FIBO), 웹(Schema.org) 등 이미 산업의 기반 인프라

**2단락 — 드론 군집에 들어오는 네 지점 (약 400자)**

각 지점을 한 문단(약 100자)으로:

**(1) 공유 어휘 (Shared Vocabulary)**
- 모든 드론이 같은 사전을 참조
- 제조사·기종이 달라도 `Drone`, `Target`, `Obstacle`, `Friendly`를 같은 의미로 인식
- "다국적 군대가 같은 군사 매뉴얼을 쓰는 것"

**(2) 시맨틱 SLAM (Semantic SLAM)**
- 단순한 점군 지도가 아니라 **의미 태그가 붙은 지도**
- "이 점은 건물 벽", "저 점은 차량", "이 영역은 통과 가능"
- 군집 전체가 의미 지도를 실시간 공유

**(3) 임무 분해 추론 (Task Decomposition)**
- "재난수색"이라는 추상적 임무를 → 영역분할 → 격자스캔 → 객체확인 → 보고로 자동 분해
- 추론 엔진이 드론별 역할을 동적으로 배정
- 한 대가 추락해도 즉시 재배정되는 자율성

**(4) 상황 인지 (Situational Awareness)**
- "온도 60도 + 가시광선 이상 + 연기 패턴 → 화재"라는 의미 규칙
- 정의된 사건이 자동으로 모드 전환 트리거: `SearchMode → RescueMode`
- 사람의 명령 없이도 상황에 맞춰 행동을 바꾼다

**3단락 — 핵심 통찰 (약 150자)**
- 좌표 공유는 **위치를 맞추는 일** — 같은 곳을 보는 것
- 의미 공유는 **인식을 맞추는 일** — 같은 것을 같다고 부르는 것
- 후자가 가능해질 때 비로소 군집은 *집합*에서 *팀*으로 바뀐다
- "드론은 좌표를 공유하는 것이 아니라 세계관을 공유한다"

## 참고할 사실 (검증된 정보)

- **온톨로지의 형식 기반**: RDF (Resource Description Framework), OWL (Web Ontology Language), W3C 표준
- **추론 엔진 예시**: Pellet, HermiT, FaCT++ (모두 학술/오픈소스)
- **시맨틱 SLAM**: 2017년 이후 활발한 연구 분야. ORB-SLAM에 시맨틱 레이어를 결합한 시도들
- **다중 로봇 임무 분해**: BDI(Belief-Desire-Intention) 에이전트 모델과 결합되는 경우가 많음

## 표현 도구

- "좌표는 위도·경도, 의미는 이름과 역할"
- "GPS가 '어디'를 답한다면, 온톨로지는 '무엇'과 '왜'를 답한다"
- "지도는 같은데, 지도 위에 그려진 세계가 같아지는 일"
- 비유: **다국적 의료팀이 SNOMED CT라는 공통 사전을 쓰는 이유** — 드론도 같은 이유로 온톨로지가 필요하다

## 주의사항

- 학술 용어를 풀어쓰는 데 정성을 들일 것. 독자는 비전공자
- "RDF/OWL"은 한 번 정도만 등장. 약어 폭격 금지
- 4가지 지점을 균등하게 다루지 말고, **공유 어휘와 임무 분해**에 무게중심
- 마지막 단락은 짧고 강하게 — 명언처럼 인용될 수 있게
